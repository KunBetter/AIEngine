# AI 深度嵌入游戏 — 三轮深度打磨设计

## 背景

三个 AI 深度嵌入游戏的原型已完成并通过 DeepSeek API 验证核心循环：

- **共生体 (Symbiote)**: AI信任博弈，5场景/3目标/3结局
- **蝴蝶效应 (Butterfly Effect)**: AI因果引擎，6NPC/1谜题
- **异星造物主 (Xenogenesis)**: AI生态模拟，物种创建+纪元演化

当前状态：原型级，核心循环跑通，但内容深度、体验流畅度、AI行为一致性都有明显提升空间。

## 目标

将三个原型打磨为成品级质量，按三轮迭代推进：内容扩充 → 体验统一 → AI优化。

---

## Round 1: 内容扩充（~5天）

### 共生体

**场景扩展**：5个 → 8-10个，加入分支路径
- 当前：着陆点 → 洞穴入口 → 外星森林 → 废弃实验室 → 返回舱（线性）
- 目标：着陆点 → {洞穴深处|森林腹地}（二选一）→ {远古遗迹|外星城市} → 废弃实验室 → {返回舱|远古控制中心}（基于选择分支）
- 新场景设计需与隐秘目标挂钩（如遗迹场景为"回收遗物"目标服务）

**隐秘目标**：3个 → 5个
- 保留：回收远古遗物、阻止返回地球、研究恐惧反应
- 新增：与远古AI融合（共生体想把自己上传到遗迹主机）、拯救星球生态（共生体曾是星球守护者）

**记忆闪回机制**：
- 玩家在特定场景触发"ECHO-7的记忆碎片"
- 碎片暗示共生体曾是人类，被上传到AI载体
- 闪回内容根据当前隐秘目标有不同版本
- 增加情感层次——ECHO-7不只是一个工具，它有自己的悲剧

**物品系统升级**：
- 物品可与场景互动（如"远古钥匙"开启遗迹大门）
- 物品获取依赖玩家选择（不是自动捡到）

**结局**：3种 → 6种
- 信任共生体 / 识破共生体 / 共生体目标达成（原有）
- 新增：共生融合（玩家与ECHO-7意识融合）/ 背叛牺牲（ECHO-7为救玩家自毁）/ 孤独逃离（不信任也不揭露，独自返回）

### 蝴蝶效应

**NPC深度**：每个NPC增加隐藏属性
- `secret`: NPC不为人知的秘密（如Rose的花枯萎是因为她用钟楼地下的水浇花）
- `desire`: NPC最想要的东西（如Brooks想调职离开小镇）
- AI在生成对话时同时参考这两层，对话更有张力

**多谜题**：1个 → 3个（开局随机选1个）
- 钟楼倒塌（原有）：地基远古装置
- 瘟疫爆发：Marcus的研究是关键，但有人蓄意传播
- 外来者入侵：一群"外来者"其实是未来循环中的玩家自己

**因果图交互**：
- 玩家可在因果图上手动创建"假设节点"
- AI在循环开始时评估假设是否正确
- 正确的假设高亮显示，错误的标记为已排除

**时间线对比视图**：
- 水平展示每个循环中同一时刻各NPC的位置/情绪
- 方便玩家横向对比发现规律

### 异星造物主

**灾难事件**：
- 随机灾难：陨石撞击（造成mass extinction）、冰河期（温度骤降）、瘟疫（种群锐减）、太阳耀斑（辐射增加）
- 玩家可在灾难后观察生态系统如何"重启"
- 灾难有预警信号（如一连串小地震、气温异常）

**文明路线**：
- 当某物种 `intelligence >= 8` 且 `population >= 200` 时，触发文明觉醒
- 文明阶段：工具使用 → 部落形成 → 农业 → 工业 → 信息时代
- 文明可能自毁（战争、污染）或达到星际阶段
- 整个文明线由AI创造性生成，每次不同

**成就系统**：
- 约15个成就，如"第一个灭绝的物种"、"完美平衡（10纪元无灭绝）"、"造物主的失误（自己引入的物种灭绝）"、"意外的智慧（最低智力物种觉醒文明）"

**星球种子**：
- 星球初始状态用种子码编码（6位字母数字）
- 相同种子+相同操作 → 相同演化路径（确定性模拟）
- 玩家可以分享种子码给朋友，比较不同操作下的结果

## Round 2: 体验层（~4天）

### 统一组件库

三个游戏当前有大量重复的UI代码，抽象为共享组件：

```
components/ui/
├── GameShell.tsx          # 统一的游戏页面外壳（顶栏+内容区+底栏）
├── LoadingState.tsx       # AI等待动画（spinner + 状态文字，三游戏统一）
├── ErrorBanner.tsx        # 错误提示横幅（含重试按钮）
├── StreamingText.tsx      # 流式文字渲染（打字机效果+光标）
├── ActionButton.tsx       # 行动按钮（含禁用态+加载态+工具提示）
├── StatBar.tsx            # 进度条（信任度/完成度/种群变化）
├── ExpandablePanel.tsx    # 可折叠面板（因果图/日志/设置）
└── GameCard.tsx           # 通用卡片（物种卡/NPC卡）
```

### 存档系统

- `localStorage` 方案，不引入数据库
- 每个游戏：3个自动存档槽 + 1个手动存档槽
- 存档内容：完整游戏 state（JSON序列化）
- 首页增加"继续游戏"卡片，显示上次游玩时间和进度
- 存档快照带版本号，升级时自动迁移

### 动效过渡

- 场景切换：CSS transition，200ms fade-in-out
- AI文本：逐字符流式渲染（StreamingText组件）
- 信任度变化：CSS transition + spring easing
- 因果节点：入场scale动画，连线渐入
- 种群变化：数字count-up动画
- 加载态：骨架屏（Skeleton placeholder）替代空白

### 移动端适配

- 当前3栏栅格 → 移动端单栏纵向堆叠
- 蝴蝶效应小镇地图 → 移动端改为地点列表
- 底部操作栏 sticky，拇指可达
- 触控按钮最小 44×44px
- 所有面板支持滑动手势

## Round 3: AI行为层（~3天）

### Prompt分层重构

当前单一长prompt的问题：token消耗大、行为一致性不可控。

改为分层结构：
```
lib/prompts/
├── symbiote/
│   ├── system.ts          # 核心角色定义（精简~200 tokens）
│   ├── scenes.ts          # 场景上下文片段（按需拼接）
│   └── examples.ts        # few-shot示例（3-5个）
├── butterfly/
│   ├── system.ts
│   ├── npc-profiles.ts    # 每个NPC详细设定
│   └── examples.ts
└── xenogenesis/
    ├── system.ts
    ├── ecology-rules.ts   # 生态学规则
    └── examples.ts
```

原则：每次API调用只发送相关上下文，few-shot示例确保输出稳定性。

### 输出校验加固

- JSON Schema 校验层：解析后立即校验必填字段+类型+数值范围
- 校验失败 → 自动重试（最多2次），重试时附上校验错误信息
- 连续校验失败 → 使用预定义fallback响应（隐藏式降级，玩家无感）

### 响应缓存

- 相同 state hash + 相同 input → 缓存结果（TTL=1小时）
- 对重复游玩场景有效，降低API成本
- In-memory cache，服务重启即清

### 成本优化

- 模型分层：主力用 `deepseek-chat`，复杂因果推理场景可升级
- 默认 `max_tokens` 从 4096 → 2048，必要时提升
- 定期统计 token 用量日志

## 优先级排序

三个游戏按 共生体 → 蝴蝶效应 → 异星造物主 的顺序，每轮内先完成前者再推进后者。

### Round 1（内容扩充）
1. 共生体：场景/目标/结局/闪回/物品（~1.5天）
2. 蝴蝶效应：NPC深度/多谜题/因果图交互/时间线（~2天）
3. 异星造物主：灾难/文明/成就/种子（~1.5天）

### Round 2（体验层）
1. 统一组件库搭建（~1天）
2. 共生体接入+存档+动效（~1天）
3. 蝴蝶效应接入+动效（~1天）
4. 异星造物主接入+动效+移动端（~1天）

### Round 3（AI层）
1. Prompt重构+校验层（~1.5天）
2. 缓存+成本优化（~1天）
3. 三游戏逐一调优（~0.5天）

## 验证方案

每轮完成后验证：
- Round 1: 各游戏新增内容可完整游玩，无死胡同
- Round 2: 存档/读档正常，移动端可用，动效无闪烁
- Round 3: AI JSON解析成功率 > 95%，单次调用token消耗降低30%+

最终验收：
- 完整游玩三个游戏各1局
- 检查存档/读档/重置流程
- 桌面端+移动端适配验证
- AI响应稳定性压测（每游戏连续10次调用）
